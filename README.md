# 🖱️ PsychoMouse

**Real-time Fatigue & Stress Detection via High-Frequency Mouse Dynamics Analysis**

> Detect cognitive fatigue using *only* a standard mouse — zero additional hardware, completely non-intrusive.

---

## Overview

PsychoMouse is a software-based system that monitors mouse cursor behavior to detect cognitive fatigue in real time. By capturing high-frequency kinematic data (1000 Hz) from a standard gaming mouse, the system applies signal processing techniques — Butterworth filtering, derivative computation, and FFT spectral analysis — to extract features that distinguish between alert and fatigued states. A machine learning classifier (SVM) achieves **73% accuracy** and **0.78 AUC-ROC** in cross-validated evaluation.

## Pipeline

```
Mouse Input ──→ Resample ──→ Butterworth ──→ Derivatives ──→ FFT ──→ ML Classify
 1000 Hz        200 Hz      4th order LP     v(t),a(t)    8-12 Hz    SVM/RF/LR
                uniform      cutoff 20 Hz     j(t)        tremor
```

### 1. Data Collection

Captures raw mouse events at high frequency using `pynput`:

- **Events recorded**: cursor position (x, y), timestamps (ms), event type (move/click/scroll)
- **Buffered writes**: flushes to CSV every 5,000 events for minimal I/O overhead
- **Fatigue self-reports**: KSS (Karolinska Sleepiness Scale, 1-9) collected every 30 minutes

### 2. Signal Processing

| Step | Method | Parameters | Purpose |
|------|--------|------------|---------|
| Resample | Linear interpolation | → 200 Hz uniform grid | Enable filtering and FFT on irregular data |
| Filter | Butterworth low-pass | 4th order, 20 Hz cutoff, zero-phase | Remove sensor jitter (>50 Hz), preserve motor signals |
| Velocity | 1st derivative | `numpy.gradient` | Cursor speed |
| Acceleration | 2nd derivative | `numpy.gradient` | Motion dynamics |
| Jerk | 3rd derivative | `numpy.gradient` | Movement smoothness (key fatigue indicator) |

### 3. FFT Spectral Analysis

Performed on idle/slow-moving segments (speed < 50 px/s) to isolate involuntary tremor from intentional motion.

| Frequency Band | Range | Significance |
|----------------|-------|--------------|
| Voluntary motion | 0–5 Hz | Intentional cursor movements |
| **Physiological tremor** | **8–12 Hz** | **Increases with cognitive fatigue** |
| Sensor noise | >20 Hz | Removed by Butterworth filter |

**Key metrics**:
- **Tremor Energy Ratio** = Power(8–12 Hz) / Total Power
- **Spectral Entropy** = Shannon entropy of normalized PSD
- Hanning window applied to reduce spectral leakage

### 4. Feature Extraction

19 features extracted from **30-second sliding windows** with 50% overlap (15s step):

| Category | Features | Description |
|----------|----------|-------------|
| Velocity | `velocity_mean`, `velocity_std`, `velocity_max`, `velocity_median` | Cursor speed statistics |
| Acceleration | `accel_mean`, `accel_std`, `accel_max` | Motion dynamics |
| Jerk | `jerk_rms`, `jerk_std`, `jerk_max` | Movement smoothness (higher = more erratic) |
| Spectral | `tremor_energy_ratio`, `spectral_entropy` | FFT-based tremor and complexity metrics |
| Path | `path_efficiency` | Straight-line / actual path distance |
| Behavioral | `pause_count`, `pause_frequency` | Pauses >500ms |
| Direction | `direction_change_std` | Movement direction variability |
| Click | `click_rate`, `click_interval_mean`, `click_interval_cv` | Click tempo and variability |

### 5. Labeling Strategy

Time-based binary labeling per session:

```
|← Alert →|←── Transition (excluded) ──→|←── Fatigued ──→|
0 min     10 min                        40 min         ~65 min
```

- **Alert**: first 10 minutes (user is freshest)
- **Transition**: 10–40 minutes (excluded from training, ambiguous)
- **Fatigued**: after 40 minutes (fatigue has accumulated)

### 6. Machine Learning

Three classifiers evaluated with Stratified K-Fold cross-validation:

| Model | Accuracy | F1 Score | AUC-ROC |
|-------|----------|----------|---------|
| Random Forest | 0.63 | 0.63 | 0.59 |
| Logistic Regression | 0.71 | 0.77 | 0.78 |
| **SVM (RBF)** | **0.73** | **0.79** | **0.78** |

All models use `StandardScaler` preprocessing and `class_weight="balanced"`.

**Top 5 most important features** (RF importance):
1. `click_rate` — click frequency decreases with fatigue
2. `velocity_median` — cursor speed changes
3. `velocity_mean` — overall movement reduction
4. `velocity_max` — peak speed reduction
5. `click_interval_cv` — click timing becomes more variable

## Quick Start

### Requirements

```bash
# Data collection
pip install pynput

# Analysis
pip install numpy pandas scipy scikit-learn matplotlib seaborn joblib

# Demo (requires Node.js)
brew install node  # Mac
# or download from https://nodejs.org/
```

### Collect Data

```bash
# Start a collection session (default: 180 min, KSS every 30 min)
python data_collection/collect_data.py --participant YourName

# Shorter session for testing
python data_collection/collect_data.py --participant YourName --duration 60 --fatigue-poll 15
```

During collection:
1. Fill in baseline KSS rating (1-9)
2. Use your computer normally
3. Respond to KSS prompts every 30 minutes
4. Press `Ctrl+C` to stop early

### Run Analysis

```bash
# Run the full pipeline: load → process → extract features → train → evaluate → visualize
python analyze.py run

# Just label all sessions
python analyze.py label-all

# Visualize a single session
python analyze.py visualize data/eason02/20260221_152030
```

Output files are generated in `results/` and `models/`.

### Run Demo

```bash
# Mac
cd live_demo
chmod +x setup_demo.sh
./setup_demo.sh
cd psychomouse-demo
npm start

# Windows
# Double-click live_demo/setup_demo.bat, then:
cd psychomouse-demo
npm start
```

Open http://localhost:3000 and choose between:
- **Live Demo** — real-time mouse tracking with signal visualization
- **Replay Demo** — recorded session playback showing Alert → Fatigued transition

## Results

### Key Findings

1. **Behavioral features dominate** — click rate and velocity statistics are more discriminative than spectral features, suggesting fatigue primarily manifests as changes in interaction tempo.

2. **SVM generalizes best on small data** — Random Forest overfits significantly (96% training accuracy vs. 63% CV), while SVM finds smoother, more generalizable decision boundaries.

3. **Butterworth filter effectively removes noise** — high-frequency sensor jitter is attenuated while preserving the 8–12 Hz physiological tremor band and voluntary motion signals.

4. **Session length affects results** — 65-minute sessions may not induce deep fatigue. Longer sessions (3+ hours) would likely improve class separation and model accuracy.

### Limitations

- Short session durations (~65 min vs. planned 3+ hours)
- Small dataset (5 participants, 6 sessions)
- Time-based labeling is an approximation of actual fatigue state
- No controlled task (free computer use introduces variability)
- Random Forest overfitting indicates need for more data

## Future Work

- **Longer sessions** (3+ hours) to capture deeper fatigue states
- **More participants** (10+) for leave-one-subject-out cross-validation
- **KSS-based labeling** using self-reports as ground truth instead of elapsed time
- **Activity segmentation** to control for different usage patterns (gaming vs. browsing vs. coding)
- **Temporal deep learning** (LSTM, 1D-CNN) for sequential pattern recognition
- **Real-time prototype** with system tray notification for fatigue alerts
